## План разработки CLI-приложения для учета финансов

### 1. Архитектура проекта

[x] Проект должен быть разделён на слои (модульная структура).
[x] Необходимо выделить слой Model — классы-модели (Transaction, Category, User).
[x] Необходимо выделить слой Service — бизнес-логика (подсчёты, аналитика, лимиты).
[x] Необходимо выделить слой Repository — работа с сохранением и загрузкой данных (файл `.txt` или `.json`).
[x] Необходимо выделить слой UI — консольное меню с циклом `while`, обрабатывающим действия пользователя.

---

### 2. Распределение функциональности

#### Блок 1: Логика и данные (Back-end)

[x] Необходимо реализовать класс `Transaction` со следующими полями: id, сумма, категория, дата, комментарий.
[x] Необходимо создать менеджер транзакций с методами `addTransaction` и `deleteTransaction`.
[x] Должен быть реализован метод подсчёта общего баланса.
[x] Должен быть реализован метод группировки расходов по категориям.
[x] Должна быть реализована проверка превышения месячного лимита.
[x] Необходимо реализовать сохранение списка транзакций в файл.
[x] Должна быть реализована загрузка данных из файла при запуске программы.

---

#### Блок 2: Интерфейс и взаимодействие (CLI)

[x] Необходимо реализовать главное текстовое меню (добавить доход, добавить расход, показать отчет и т.д.).
[x] Должна быть реализована проверка корректности ввода (валидация через `try-catch` или `Scanner.hasNextDouble()`).
[x] Вывод отчетов должен быть аккуратно отформатирован через `System.out.printf`.
